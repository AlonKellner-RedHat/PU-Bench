## Why Stratify by Dataset?

Different datasets have wildly different baseline performance levels:
- **Text** (20News, IMDB): Lower AUC (~0.70-0.85) due to high-dimensional sparse features
- **Tabular** (Mushrooms, Spambase): Medium AUC (~0.85-0.95) with clear discriminative features
- **Vision** (MNIST, FashionMNIST): High AUC (~0.95+) on simple digit/fashion classification

By showing results **per dataset**, error margins reflect only **seed variance** (tight), not dataset heterogeneity (large).

This reveals:
1. Which methods are **robustly better** (win across datasets)
2. Which methods are **modality-specific** (excel on text but not vision)
3. **Statistical precision** - tighter confidence from separating variance sources

| Mushrooms | 0.9731 | 0.0571 | 75 |
| Spambase | 0.8429 | 0.1391 | 75 |

**Aggregated across datasets:**
- Mean: 0.8489
- Std (seed variance only, within-dataset): ~0.01-0.03 (see above)
- Std (dataset variance, cross-dataset): 0.1490

**Ratio**: Dataset variance is ~7.5× larger than seed variance!

This shows why stratifying by dataset provides much tighter error margins.
